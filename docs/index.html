Что такое LiseryMenu

LiseryMenu — плагин для Paper / Spigot 1.21–1.21.10, вдохновлённый DeluxMenus. Мы сохранили знакомый формат YAML, добавили современные улучшения и интеграции, а также сделали акцент на лёгкой настройке и производительности.

🚀 Быстрый старт

  • /lm create menu — создать файл меню
  • файлы в /plugins/LiseryMenu/menus
  • /lm reload — перечитать YAML + команды

🧠 Современная логика

  • if / elif / else со списками условий
  • поддержка PlaceholderAPI и Vault
  • display_condition для разных состояний предмета

🔗 Интеграции

  • ItemsAdder: ia:namespace-item
  • Oraxen: oraxen-item
  • Nexo: nexo-item
🚀 5 минут до первого меню

Быстрый старт

1. Установка

  1. Скачайте LiseryMenu-1.0.jar.
  2. Поместите в /plugins.
  3. Перезапустите сервер — появится папка LiseryMenu.
  4. В папке menus будут примеры.

2. Первое меню

  1. /lm create shop — создаёт shop.yml.
  2. Заполните предметы в файле.
  3. /lm reload — обновите меню.
  4. Команда из open_command активируется сразу.

3. Стартовый YAML

title: "&6Примерное меню"
size: 27
open_command: "example"

items:
  info:
    slot: "13"
    material: PLAYER_HEAD
    skull_owner: "%player_name%"
    display_name: "&eПривет, %player%!"
    left_click_actions:
      - "[message] <#ffaa00>Меню открыто!"
      - "[close]"
📜 Управление

Команды и права

Команда Описание Permission
/lm create <name> Создаёт файл menus/<name>.yml и регистрирует команду меню. liserymenu.create
/lm reload Перезагружает все меню и заново регистрирует команды. liserymenu.reload
/lm list Показывает загруженные меню и привязанные open_command. liserymenu.list
Совет Команды меню появляются и исчезают автоматически при загрузке/удалении YAML. Отдельный рестарт сервера не требуется.
🧩 Структура YAML

Описание меню

Основные поля: title, size, open_command, hide_items. Ветка items описывает содержимое меню. Поддерживаются форматы слотов "13", "0,1,2", "0-8,18-26". 

title: "&8Арсенал"
size: 54
open_command: "armory"
hide_items: true

items:
  banner:
    slots: "0-8,45-53"
    material: GRAY_STAINED_GLASS_PANE
    display_name: " "

  category_weapons:
    slot: "20"
    material: DIAMOND_SWORD
    display_name: "&cОружие"
    left_click_actions:
      - "[open_menu] armory_weapons"

  vip_button:
    display_condition:
      if: "permission: donate.vip"
      then:
        slot: "10"
        material: GOLD_BLOCK
        display_name: "&6VIP Статус"
        lore:
          - "&7Спасибо за поддержку!"
      else:
        slot: "10"
        material: GOLD_BLOCK
        display_name: "&eПолучите VIP"
        right_click_actions:
          - "[message] <#ffaa00>Поддержите сервер, чтобы открыть VIP!"
🔐 Проверки

Условия if / elif / else

Условия можно задавать строкой или списком. Список в if: означает проверку по схеме AND. Конструкция elif принимает список блоков с if и then.

Тип Синтаксис Пример
Права permission: <node> permission: armory.vip
Экономика %vault_eco_balance% >= 1000 if: "%vault_eco_balance% >= 5000"
Предметы has_item: material amount has_item: ia:lisery-ruby 3
Комбинированное условие if: + список if: ["permission: donate.vip", "%player_level% >= 30"] 
left_click_actions:
  if:
    - "%vault_eco_balance% >= 1000"
    - "permission: donate.vip"
  then:
    - "[console] eco take %player_name% 1000"
    - "[message] <#9cff6b>Покупка выполнена!"
  elif:
    - conditions:
        - "permission: donate.vip"
      then:
        - "[message] <#ffaa00>Накопите ещё немного средств."
  else:
    - "[message] <#ff5547>Необходим VIP и 1000$."

🔌 Интеграции

Поддерживаемые плагины

PlaceholderAPI

Автоматически подключается, если установлен. Используйте плейсхолдеры в названиях, описаниях, условиях и действиях.

display_name: "&eБаланс: %vault_eco_balance%"

ItemsAdder

Формат: ia:namespace-item. Внутри автоматически преобразуется в namespace:item.

material: "ia:lisery-radiant_sword"

Oraxen / Nexo

Используйте oraxen-item_id и nexo-item_id. Проверка has_item также поддерживает эти форматы.

❓ FAQ

Частые вопросы

Нужно ли PlaceholderAPI / ItemsAdder?

Нет. Они подключаются как softdepend: плагин работает без них, но включает дополнительные функции при наличии.

Как скрыть инвентарь игрока?

Установите hide_items: true. Предметы игрока временно уберутся и вернутся при закрытии меню.

Shift-клики

Начиная с версии v1.3 поддерживается shift_left_click_actions. Для дополнительных сценариев используйте display_condition или отдельные предметы.
